# Guest-side runtime

There is no guest source to build here. The in-VM pieces live in two places:

- **The connect-back agent** (`pkg/main.lua`) and the entire MicroNT base system
  are **baked into the template `initrd.zip`** (the nt365 build ships them). The
  agent boots, brings up DHCP, dials back to the host on the per-instance port,
  and runs each Lua chunk the host sends. We do **not** inject or override it.

- **The stager** — the Lua chunk the host sends over the connect-back socket — is
  generated by [../orchestrator/stager.ts](../orchestrator/stager.ts). It
  `require()`s the user's module (baked into the initrd `pkg/` dir as a STORED
  zip) and frames the JSON result back over the socket.

The only contract a user writes against is the package entrypoint:

```lua
-- a package required as `hello` returns a function(args)
return function(args)
  return { greeting = 'hello ' .. (args[1] or 'world') }
end
```

See [../../examples/hello/init.lua](../../examples/hello/init.lua) and the
"Instance lifecycle" section of [../../OUTLINE.md](../../OUTLINE.md).
